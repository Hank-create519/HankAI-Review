# 瀚海贸易 Agent 系统 · 完整方案（已修复审查问题）

> 一人副业，公司效率

---

## 一、定位

一个人 + 这套系统 = 选品/定价/上架/采购/物流/客服/财务/风控 八个岗位的产出。

你只做两件事：定战略方向，处理系统搞不定的异常。

### 二、核心效率指标

| | 纯人工 | 人+本系统 |
|---|---|---|
| 同时管理商品数 | 30个 | 300个 |
| 日出单处理能力 | 10-20单 | 不限（自动） |
| 客服响应 | 有空才回 | L1秒回/L2草稿待确认 |
| 选品扫描量 | 日刷几十个 | 日扫上千个 |
| 需要的人力 | 你一个人忙不过来 | 你一个人够用 |

### 三、系统架构（四层）

```
事件监听层（24h运行）
  新订单 │ 物流更新 │ 客户消息 │ 价格波动 │ 定时扫描
    │
    ▼
编排引擎
  事件→匹配工作流→调度Agent→监控执行→异常升级
    │
    ▼
Agent池（8个自治Agent）
  选品工 │ 定价工 │ 上架工 │ 采购工
  物流工 │ 客服工 │ 财务工 │ 风控工
  每个Agent：主路径→备选路径→降级→升级人工
    │ 关键决策送审
    ▼
瀚海审查系统（L2交叉审核）
  3AI辩论 → 绿灯自动过 / 黄灯标记继续 / 红灯拦截
    │
    ▼
黑板（数据库）
  商品库 │ 供应商库 │ 订单库 │ 利润数据 │ 操作日志
```

### 四、8个Agent详细定义

#### 选品工（Scout）
日扫1688/拼多多/义乌购，对比eBay/Shopee/Amazon售价，发现利润>30%的商品。

| 项目 | 内容 |
|---|---|
| 触发 | 定时自动运行（每天1次）/ 用户指定品类 |
| 输入 | 品类关键词、预算上限、目标平台 |
| 逻辑 | 爬取国内价格 → 比对海外售价 → 计算价差率 → 风控扫雷 |
| 输出 | 候选商品写入商品库（含粗算利润率） |
| 自主程度 | 全自动，利润>30%+无风险标记→直接入库候选 |
| 备选路径 | 1688搜不到→切拼多多。eBay价格缺失→切Shopee参考 |
| 升级条件 | 指定品类搜不到任何结果、疑似品牌侵权 |

**选品利润率 ≠ 定价利润率**：选品工做的是粗算（海外参考价 vs 1688价），定价工会逐项精确计算。当定价工算出的利润率比选品工低了15个百分点以上时，系统自动标记该商品为"利润率需复核"，不直接丢弃，但也不自动上架——写入黄灯区供你查看。你决定是接受调整后的利润率继续，还是放弃该品。

#### 定价工（Pricer）
逐项计算总成本，给出建议售价和预期利润。

成本公式：
```
总成本 = 商品进价 + 国内运费(8元/kg) + 国际运费(按重量查价)
       + 平台佣金(eBay 13.25%) + PayPal手续费(4.4%+$0.30)
       + 关税预估(目的国税率×申报价值) + 退货预留(售价×5%)

建议售价 = 总成本 × (1 + 目标利润率)
```

| 项目 | 内容 |
|---|---|
| 触发 | 商品库出现新候选商品 |
| 输入 | 商品信息（成本价、重量、尺寸）+ 目标平台 |
| 输出 | 写入黑板：建议售价区间、成本拆解、预期利润率、选品粗算vs精确利润率偏差 |
| 自主程度 | 全自动，利润率达标即定 |
| 降级策略 | 运费查不到→用同类历史运费均值估算，标记"估算值"；关税不确定→按默认税率×1.2保守计算 |
| 升级条件 | 关键数据全缺失、算出来利润<10% |

**冷启动调价**：新listing（无销量无评价的前14天），定价工自动将建议售价下调10%作为促销价，帮助获取初始销量。14天后自动恢复原价。

#### 上架工（Lister）
生成海外平台listing并发布。

| 项目 | 内容 |
|---|---|
| 触发 | 定价完成，商品状态为approved |
| 阶段1（当前） | 生成结构化listing草稿（标题+描述+关键词+图片建议），用户手动复制上架 |
| 阶段2（远期） | 浏览器自动化模拟人工上架操作（标注：有封号风险，需谨慎启用） |
| Listing规则 | 标题60-80字符「核心词+属性词+场景词+卖点词」；描述Markdown含规格表；关键词提取高搜索量长尾词 |
| 自主程度 | 自动生成并发布（阶段2），文案可后续调优 |

#### 采购工（Buyer）
检测到订单→检查价格库存→向供应商下单。

| 项目 | 内容 |
|---|---|
| 触发 | 订单库出现新订单 |
| 阶段1 | 生成采购清单（供应商+商品+数量+客户地址），用户手动下单 |
| 阶段2 | 浏览器自动化模拟1688下单 |
| 自主程度 | 价格不变或涨价≤15%→自动下单；涨价>15%→重算利润，仍OK自动继续 |
| 降级策略 | 原供应商涨价>15%→自动搜索替代供应商。**搜索上限：最多搜索3个替代供应商。** 对每个替代供应商自动调用定价工验算利润，任意一个利润达标即自动切换。3个都不达标→停止搜索，直接升级通知你（附带：原供应商现价、3个替代供应商的价格和利润、建议下一步）。断货时同样适用此规则。 |

#### 物流工（Tracker）
追踪全链路物流，自动通知客户。

| 项目 | 内容 |
|---|---|
| 触发 | 采购完成，拿到物流单号 |
| 逻辑 | 定时查询物流状态→更新黑板→状态变化自动通知客户→签收触发售后跟进 |
| 告警 | 超48h无更新→通知你；延迟>3天→生成客户安抚模板；包裹丢失→计算补发vs退款成本对比 |
| 自主程度 | 全自动追踪+通知 |
| 升级条件 | 包裹丢失、客户投诉未收到 |

#### 客服工（Support）
处理客户咨询，分级响应。

| 级别 | 场景 | 处理方式 |
|---|---|---|
| L1 | 物流查询（"我的东西到哪了"） | 自动从黑板取物流状态，自动回复 |
| L2 | 产品质量问题 | **自动生成回复草稿 + 附带解决方案选项 → 推给你确认后发送。不自动发送。** |
| L3 | 纠纷/退款威胁/法律威胁 | 不自动回复，立即打包订单历史+建议方案升级给你 |

| 触发 | 客户消息事件 |
|---|---|
| 熔断 | 同日连续3笔纠纷→自动暂停L2草稿生成，所有客服请求直接升级 |

**L2设计原因**：自动发送退款/赔偿承诺存在法律风险，一旦措辞不当可能构成对你不利的承诺。改为生成草稿+方案选项，你看一眼点确认就发送，安全且不费力。

#### 财务工（Finance）
自动记账，定期出报表。

| 项目 | 内容 |
|---|---|
| 触发 | 每笔订单完成后自动记录 / 用户要求查看报表 |
| 输出 | 日/周/月利润表，按平台/品类/时间段分类统计 |
| 税务提醒 | 月度销售额接近起征点时自动预警 |
| 自主程度 | 全自动 |

#### 风控工（Risk）
每个Agent行动前扫雷。

| 检查维度 | 具体检查项 | 处理 |
|---|---|---|
| 品牌侵权 | 关键词匹配Apple/Samsung/Nike等→高风险 | 命中即拦截 |
| 禁运品类 | 液体/电池/粉末 | 标记物流限制 |
| 供应商信用 | 评分<4.5→黄灯，<4.0→红灯；经营<1年→黄灯 | 红灯拦截 |
| 价格异常 | 明显低于市场价50% | 标记可疑 |
| 客户异常 | 地址高危地区、有纠纷历史 | 标记，触发审核 |
| 平台合规 | 各类目特殊要求、违禁词 | 上架前检查 |

### 五、事件驱动的工作流

#### 核心工作流：订单事件（全自动）

```
eBay新订单检测
  │
  ▼
订单解析 → 提取商品SKU、客户地址、金额，写入订单库
  │
  ▼
风控工扫描 → 地址/金额/客户历史 正常→继续，异常→升级
  │
  ▼
采购工执行 → 访问供应商链接 → 检查价格库存
  │ 正常 → 自动下单
  │ 涨价≤15% → 重算利润，OK自动继续
  │ 涨价>15% → 搜替代供应商(最多3个)
  │   → 找到利润达标→自动切换并下单
  │   → 3个都不达标→升级给你
  ▼
物流工接管 → 获取单号 → 注册追踪 → 发发货通知给客户
  │
  ▼
物流持续监控 → 每次更新自动通知客户 → 签收触发售后 → 财务记账
```

#### 其他工作流

| 事件 | 自动执行 |
|---|---|
| 物流更新 | 更新黑板→通知客户→异常告警 |
| 客户消息 | 分类→L1自动回复/L2生成草稿等你确认发送/L3升级 |
| 定时扫描 | 选品→定价→自动上架候选 |
| 价格波动 | 重算利润→OK继续/不OK暂停 |

### 八、冷启动机制

系统从0到1需要初始推动，内置冷启动引导：

**初始化向导（首次运行）**：
1. 你选择目标品类（如：手机配件、宠物用品、家居小物）
2. 选品工首次扫描该品类，产出候选商品列表
3. 你勾选第一批商品（建议5-10个起步）
4. 定价工逐项定价，风控工扫雷
5. 上架工生成listing草稿
6. 你手动上架（首批建议亲手上，熟悉流程）

**冷启动促销策略（前14天自动执行）**：
- 定价工对新listing自动降价10%
- 第15天自动恢复原价
- 目的：用低价换初始销量和评价，突破新listing曝光瓶颈

**前10单人工陪跑**：
- 首批订单建议你亲自盯一下，确认全流程（采购→物流→客户收货）通畅
- 10单跑通后，开启全自动模式

### 九、你的操作界面

**日常**：打开仪表盘看一眼——今日订单、利润、异常升级数量。不用动手。

**被通知时**：系统弹出升级（断货找不到替代、客户要退款等），查看方案，选一个。L2客服草稿同理，看一眼点确认即发送。

**定期**：看财务报表，决定是否调整品类或利润率目标。
