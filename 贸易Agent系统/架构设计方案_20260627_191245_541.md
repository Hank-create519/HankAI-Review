### 4.2 各Agent自治决策权限

| Agent | 可自主决策 | 不可自主决策（必须升级） |

### 5.3 当前方案：轮询 + 状态机

不依赖外部API也能跑。核心逻辑：

```

  1. 抓取平台订单列表页
  2. 对比黑板已有订单ID
```

## 六、瀚海审查系统的接入点

审查系统作为L2审核引擎，在以下节点自动送审：

| 送审节点 | 送审内容 | 审查系统3AI辩论 | 结论处理 |
|---|---|---|---|
| 选品产出后 | 候选商品清单+原始数据 | 逻辑验证价格合理性+事实验证供应商资质+综合风险评估 | 绿灯自动过/黄灯标记续/红灯拦截 |
| 定价产出后 | 定价方案+成本拆解 | 逻辑验证公式+事实验证运费关税+综合判断定价竞争力 | 同上 |
| 采购前 | 供应商信息+订单详情 | 逻辑验证是否最优采购方案+事实验证供应商当前状态+综合判断采购风险 | 同上 |
| 纠纷处理 | 纠纷详情+历史记录 | 3方辩论责任归属+调解方案 | 自动执行或升级人工 |

审查系统返回结构化结论：

```json
{
  "verdict": "green",  // green / yellow / red
  "confidence": 0.92,
  "consensus": ["价格数据可验证", "供应商信誉良好"],
  "concerns": [],
  "suggestions": []
}
```

- **green**：贸易系统自动继续，不通知用户
- **yellow**：贸易系统自动继续但标记日志，事后可查
- **red**：贸易系统暂停，打包上下文通知用户

## 七、人工的唯一入口：异常升级

正常路径下，你什么都不用管。升级通知只在以下情况触发：

| 升级原因 | 携带信息 | 你的操作 |
|---|---|---|
| 采购涨价>15%且无替代 | 原价/现价/利润影响/已搜索的替代品 | 接受涨价 / 退款客户 / 换品类 |
| 商品断货且无替代 | 商品信息/客户订单金额 | 退款 / 让客户等 |
| 客户发起纠纷 | 纠纷详情/订单历史/建议方案 | 选择方案并回复 |
| 疑似品牌侵权 | 商品信息/侵权依据 | 下架 / 继续（风险自担） |
| 触及税务阈值 | 累计销售额/税务提醒 | 自行处理 |

每次升级通知都是「问题描述 + 系统已尝试的方案 + 建议下一步」，你选一下就行，不用从头分析。
